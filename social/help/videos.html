ビデオ

VideoCloudアカウントからYouTube、Facebook、Twitterに動画を公開する。

ビデオを公開する前に、ソーシャルアカウントの宛先を作成する必要があります。左側のナビゲーションで、行き先.

ビデオレンダリング

ソーシャルプラットフォームにビデオを配信するには、ビデオには次のうち少なくとも1つが必要です。

  • MP4レンディション
  • ビデオマスター

標準の取り込みプロファイルの1つを使用してアップロードされたビデオは、デフォルトでマスターを保存します。

BrightcoveSocialがソーシャルプラットフォームにビデオを公開すると、次のようになる。

  • ソーシャルプラットフォームのルールに沿った最高品質のレンディションを配信しようとする試み
  • ソーシャルプラットフォームに送信されたレンディションが拒否された場合、別のレンディションを送信しようとする試みはありません。

ビデオのランク付けルール:

  • MP4アセットは他のどのビデオコンテナよりも優れています。
  • 符号化レートが高いビデオの方が優れていると考えられます。
  • 高解像度のビデオの方が良いと考えられます。
ソーシャルメディアへの公開
  1. 公開するビデオを選択します。
  2. クリック公開.
  3. 公開先のソーシャル デスティネーションを選択するか、新しい目的地を追加.
  4. 公開する各宛先のビデオメタデータプロパティを確認します。
  5. クリック宛先に公開.

ビデオを手動でパブリッシュすることは1回限りの操作であり、これらのビデオは最初のアップロード後にSocialモジュールからメタデータの更新を受信しません。

YouTubeのプロパティを設定

デフォルトでは、最長15分のビデオをYouTubeにアップロードできます。

ビデオサムネールソース

  • ユーチューブ- YouTube がサムネイルを選択します
  • ビデオクラウド- Video Cloud のサムネイル画像を YouTube のサムネイルとして使用する
  • カスタム画像- 使用するカスタム サムネイル画像を参照して選択する

チェック中すべての目的地に使用は、現在パブリッシュされているすべての宛先に対してこのサムネイルを使用します。

Studio でサムネイル画像を更新しても、YouTube 側のサムネイル画像は更新されません。

PostInfoプロパティ

  • 題名- ビデオのタイトル (デフォルトは Video Cloud のタイトル); YouTube は最大 100 文字をサポートしています
  • 説明- ビデオの説明 (デフォルトは Video Cloud の短い説明); YouTube は最大 5000 文字をサポートしています
  • YouTube カテゴリ- 動画のコンテンツ カテゴリ
  • タグ- ビデオのタグ (デフォルトは Video Cloud タグ);タグごとに最大 30 文字を使用でき、テキスト フィールドは最大 500 文字を受け入れます

スケジュールプロパティ

  • すぐに公開- 動画をすぐに公開するには
  • 公開日のスケジュール- ビデオの公開日時を選択するには
スケジュールの日付と時刻は、動画がYouTubeに送信されたときに一度だけYouTubeに送信されます。スケジュールの日付や時刻を変更する必要がある場合は、YouTube で変更する必要があります。

キャプションのプロパティ

  • Video Cloud キャプションを使用する- Video Cloud でビデオに関連付けられたキャプションを使用する
  • キャプションなし- キャプションは YouTube に送信されません
  • キャプションをアップロードする- .VTT または .SRT 形式のキャプション ファイルを選択します
Socialを使用して字幕を切り取ったビデオには字幕はありません。クリップされたビデオに新しいキャプションをアップロードする必要があります。

可視性プロパティ

  • 公衆- 誰でもビデオを見ることができます
  • 未公開- 動画へのリンクを知っている人なら誰でも視聴できます
  • プライベート- ビデオは、あなたとあなたが選択したユーザーのみが見ることができます

選択中ビデオが YouTube に投稿されたときにサブスクライバーに通知するビデオがアップロードされたという通知をサブスクライバーに送信する場合があります

ライセンスプロパティ

  • 標準YouTubeライセンス
  • クリエイティブコモンズ - 帰属

Facebookのプロパティを設定

Facebookは .gif画像を許可しません。お客様の動画が .gifをサムネイル画像に使用している場合、Facebookへの公開は失敗します。

ビデオサムネールソース

  • フェイスブック- Facebook がサムネイルを選択します
  • ビデオクラウド- Video Cloud のサムネイル画像を Facebook のサムネイルとして使用する
  • カスタム画像- 使用するカスタム サムネイル画像を参照して選択する

チェック中すべての目的地に使用は、現在パブリッシュされているすべての宛先に対してこのサムネイルを使用します。

Studioでサムネイル画像を更新しても、Facebook側のサムネイル画像は更新されません。

PostInfoプロパティ

  • 題名- ビデオのタイトル (デフォルトは Video Cloud のタイトル); Facebook は 65 文字を超えると切り捨てられます
  • 説明- ビデオの説明 (デフォルトは Video Cloud の短い説明)
  • タグ- リストから Facebook タグを選択します。 Facebookがタグを管理していることに注意してください
  • フェイスブックのカテゴリー- ビデオのコンテンツ カテゴリ

スケジュールプロパティ

  • すぐに公開- 動画をすぐに公開するには
  • 公開日のスケジュール- ビデオの公開日時を選択するには
スケジュール日時が Facebook に送信されるのは、動画が Facebook に送信されるときの 1 回だけです。スケジュール日や時刻を変更する必要がある場合は、Facebookで変更する必要があります。

キャプションのプロパティ

  • Video Cloud キャプションを使用する- Video Cloud でビデオに関連付けられたキャプションを使用する
  • キャプションなし- キャプションは Facebook に送信されません
  • キャプションをアップロードする- .VTT または .SRT 形式のキャプション ファイルを選択します
Facebook では、キャプション言語には、言語に加えて国を含める必要があります。例:en-US または en-UK。
Socialを使用して字幕を切り取ったビデオには字幕はありません。クリップされたビデオに新しいキャプションをアップロードする必要があります。

ターゲットプロパティ

  • オーディエンス最適化の興味- あなたの投稿を楽しむ可能性が最も高い人々の興味を説明するタグを追加します
  • 場所の制限- 投稿を表示する国を追加または除外する

資産の収益化

  • この宛先への投稿のビジネス パートナーは誰ですか? - ブランド コンテンツ投稿のビジネス パートナーを追加します。入力すると、一致するビジネスのリストが表示されます
  • ビジネスパートナーが投稿を後押しできるようにする

権限プロパティ

[権限] オプションを表示するには、Facebook アカウントが Facebook の権利マネージャにアクセスできる必要があります。
  • 一致するコンテンツの検出- Facebook 著作権管理を有効にする場合にチェックします
  • 所有権の指定- ビデオ、オーディオ、またはその両方の著作権所有権を指定する
  • 国の選択- 選択する世界的に、国を含めるまた国を除外
  • 一致ルールを適用- ユーザーが一致した動画をアップロードしたときに発生する自動アクションを決定します。マッチルールはFacebook内で管理されます
  • 許可を与える- 特定の人、ページ、または Instagram アカウントに、あなたのコンテンツに一致する動画を投稿する権限を与えることを許可します

配布プロパティ

  • ニュースフィードに表示- ニュースフィードにビデオを表示
  • 埋め込みを許可- サードパーティ サイトへの動画の埋め込みを許可する
  • シークレット動画として追加- 動画は直接 URL 経由でのみアクセスでき、Facebook でユーザーを検索することはできません
  • リアクション、コメント、共有をオフにする- いいねやコメントを許可しない

プロパティの追跡

  • 空白のままにする
  • Video Cloudの 動画IDを使用
  • 存在する場合、Video Cloudの リファレンスIDを使用

クリップの作成

VideoCloudアカウントの既存のビデオからビデオクリップを作成します。

クリップは、ソーシャルメディアに投稿できる短いティーザービデオとして使用できます。

  1. クリップするビデオを選択します。
  2. クリッククリップを作成.
  3. コントロールをドラッグして、クリップの持続時間を定義します。
  4. 矢印キーを使用してフレームごとに移動します。
  5. クリック継続する.

ビデオのダウンロード

ビデオURLをコピーするか、ビデオソースファイル（最高品質のMP4レンディション）をダウンロードできます。

DRMが有効になっているビデオはダウンロードできません。
  1. ダウンロードするビデオを選択します。
  2. クリックダウンロード.
  3. 動画URLをコピーするか、またはリンクをクリックして、ブラウザーで動画を開きます。
  4. ソースファイルをローカルに保存するには、動画を右クリックします。
  5. クリック近い.

レンディションと画像のURLは固定されていないことを理解することが重要です。

この URL は、パブリックソーシャルプラットフォームの投稿に貼り付ける、長期間有効なパブリックアクセス可能な URL であることを意図していません。

メディアでの表示

ビデオURLをコピーするか、ビデオソースファイル（最高品質のMP4レンディション）をダウンロードできます。

DRMが有効になっているビデオはダウンロードできません。
  1. 表示するビデオを選択します。
  2. クリックメディアで見る.
  3. 選択したビデオの詳細が表示され、メディアモジュールが開きます。
ステータスの確認

ソーシャル メディアに公開された動画が表示されますステータスを確認の中に状態桁。

  1. クリックステータスを確認ビデオ用。
  2. ステータスとともに、ビデオが公開されたソーシャル宛先のリストが表示されます。
360°ビデオの公開

YouTube、Facebook、BrightcoveWebPlayer、およびBrightcoveSDKは、360°ビデオ再生をサポートしています。

  1. ビデオをVideoCloudにアップロードします。詳細については、を参照してください。 360° ビデオの操作.
    Socialモジュールを使用して360°ビデオを同期する場合は、マスターファイルを保存するように、ビデオの取り込みに使用する取り込みプロファイルを設定する必要があります。
  2. [Social]モジュールに戻ります。
  3. ビデオを選択します。
  4. クリック公開.
  5. 公開先のソーシャル デスティネーションを選択するか、新しい目的地を追加.
  6. 公開する各宛先のビデオメタデータプロパティを確認します。
  7. クリック宛先に公開.
360°ビデオの再生は、特定のブラウザに限定されています。
ビデオを手動でパブリッシュすることは、自動同期とは無関係です。ビデオに自動同期でパブリッシュするタグが付いている場合は、ビデオを手動でパブリッシュしてもパブリッシュされます。
