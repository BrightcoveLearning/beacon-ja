    一般的な構成設定

    このトピックでは、Brightcove Beacon でアプリを作成または変更するときに使用される一般的な設定について説明します。

    概要

    このトピックは、Brightcove Beacon アプリに関連する共通の設定をまとめたものです。これらの設定は、Brightcove Beacon の特定のタスクの実行に関連する他のドキュメントに記載されています。これには、次のタブセクションが含まれます。

    • レイアウト
    • ページ
    • チャネル
    • 映画
    • プレイリスト
    • プログラム/シリーズ
    • イベント
    • コマース/パッケージ/ストリーム/高度

    権利とスケジュール

    [ 権利とスケジュール ] セクションでは、アプリの再生権限を定義し、公開スケジュールを設定できます。

    ページを追加または編集するときの例を次に示します。

    この表には、[ 権限とスケジュール ] タブに表示されるフィールドが含まれています。

    フィールド 必須 説明
    ステータス はい 次のいずれかのタイプを選択します。
    • 公開済み
    • 未公開
    権利タイプ はい チャンネルが広告をサポートする場合は、次の値から指定します。
    • 無料
    • 広告-AVOD (広告ビデオ・オン・デマンド)
    • 広告-SVOD (サブスクリプションビデオオンデマンド)
    • 継承-これは、今シーズンが属するシリーズから権利が継承されることを意味します。
    権利開始日 はい アプリの開始日時を設定します。(また、開始時刻 )
    権利終了日 はい アプリの終了日時を設定します。(また、終了時刻 )
    公開するデバイス はい 公開するデバイスを追加します。[ Select/Unselect All Devices ] を選択して、使用可能なデバイスのリストを表示します。
    許可された場所 はい

    ユーザーがコンテンツを表示できるようにする地理的場所を選択します。[ 許可された場所 ] ボックスの内側をクリックして、場所の一覧を表示します。制限がない場合は、 World と入力します。

    許可された場所を 1 つ以上指定しないと、アプリから次のメッセージが表示されることがあります。
    拒否された場所 いいえ ユーザーにコンテンツを表示させたくない地理的場所を選択します。[ 拒否された場所 ] ボックスの内側をクリックして、場所の一覧を表示します。
    最大デバイス数 いいえ アカウントのアクティブな接続デバイスの最大数を設定します。整数値です。
    最大ストリーム数 いいえ 同じアカウントから視聴できるストリームの最大数を設定します。整数値です。

    最大ストリームを4に、最大デバイスを6に設定する例：つまり、4 人がビデオをストリーミング（ストリーム）できますが、6 人接続（デバイス）を持つことができます。

    テキストデータ

    [ テキストデータ ] セクションでは、表示可能なテキストを追加できます。これは、アプリに表示される名前です。

    ページを追加または編集するときの例を次に示します。

    このテーブルには、[ テキストデータ ] タブに表示されるフィールドが含まれています。

    フィールド 必須 注
    タイトル はい アプリに表示されるタイトル。定義した各言語の入力フィールドがあります。

    通常は、[ 基本データ ] タブで名前の入力を求められます。この名前は相互参照用ですが、[ テキストデータ ] タブの [ タイトル ] はアプリに表示されるものです。

    テキスト以外のデータ

    [ 非テキストデータ ] セクションでは、アプリの背景画像とアイコン画像をアップロードできます。

    ページを追加または編集するときの例を次に示します。

    このテーブルには、[ 非テキストデータ ] タブに表示されるフィールドが含まれています。定義した言語ごとに画像をアップロードできます。

    フィールド 必須 注
    アイコン はい ページを表す画像を指定します。
    ホバーアイコン はい この画像は、ユーザーがアイコン画像の上に置いたときに表示されます。
    水平背景 はい ユーザーデバイスが横長モードのときに表示される背景画像。
    垂直背景 はい ユーザーデバイスがポートレートモードのときに表示される背景画像。

    アップロードできる最大サイズは 500 KB です。
    Brightcove Beacon 内での画像のアップロードの詳細については、「画像とその仕様」ドキュメントを参照してください。
