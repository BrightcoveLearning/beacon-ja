API 認証

基本認証を有効にしていない限り、Brightcove API を呼び出すにはアクセス トークンが必要です。プレイヤー設定.

ここでは、アプリケーションを登録することによって、各アクセストークンの範囲を定義できます。

アプリケーションの表示
  • 名前- このリンクをクリックして、アプリケーションの詳細を表示および編集します。
  • 日付発行- アプリケーションが作成された日付。
  • 消去- このボタンをクリックして、アプリケーションを削除します。
新規アプリケーションの登録
  1. クリック新しいアプリケーションを登録する.
  2. フィールド値の追加/選択:
    • 名前- アプリケーションの名前
    • 説明- アプリケーションの説明。
    • 承認するアカウントを選択- アクセス トークンを作成するアカウントを 1 つ以上選択します。
    • 公開された Brightcove API - このアプリケーションで有効にする API と権限を選択します。セキュリティ上の理由から、権限の範囲を制限することをお勧めします。
  3. クリック保存.
アプリケーションの編集
  1. アプリケーション名のリンクをクリックします。
  2. フィールド値を編集します。
    • 名前- アプリケーションの名前
    • 説明- アプリケーションの説明。
    • 承認するアカウントを選択- アクセス トークンを作成するアカウントを 1 つ以上選択します。
    • 公開された Brightcove API - このアプリケーションで有効にする API と権限を選択します。セキュリティ上の理由から、権限の範囲を制限することをお勧めします。
  3. クリック保存、キャンセル、 また消去.
